# 继续优化这份点灯算法代码的任务说明（数学逻辑版）

## 一、当前算法主线（先统一理解）

这份代码当前的数学主线可以概括为：

1. 先递推出当前 `n` 对应的

   * `f(x)`：模多项式 / 特征多项式
   * `c(x)`：按钮矩阵对应的多项式
   * `y`：右端向量（翻转向量最后一行）

2. 用扩展欧几里得求
   [
   s(x)f(x)\oplus q(x)c(x)=g(x)
   ]
   其中

   * `g(x)=gcd(f,c)`
   * `q(x)` 是后续求解要用的系数多项式

3. 计算
   [
   z=q(H)y
   ]

4. 若 `g=1`，则可逆，第一行解直接就是 `x=z`

5. 若 `g≠1`，则当前代码会继续构造
   [
   D=g(H)
   ]
   再通过回代 / 反向叠加法解
   [
   Dx=z
   ]

6. 最后由第一行解 `x` 递推出整盘按钮矩阵 `t`

---

## 二、已经确认成立的关键数学关系

### 1. `p` 可以省去，用 `c` 直接代替

原先代码里有
[
p(x)=c(x)\oplus f(x)
]
又因为
[
f(H)=0
]
所以
[
p(H)=c(H)
]

因此：

* 以后不必显式计算 `p`
* 直接用 `c` 参与 `gcd(f,c)` 和后续求 `q,g`
* 这一步已经是严格等价，不是近似替换

---

### 2. `c(x)=f(x+1)`

这一点非常重要。
也就是说，当前要做欧几里得的两个多项式，本质上是：

[
f(x),\quad f(x+1)
]

因此：

* `c` 不是独立的神秘对象
* `g=gcd(f,c)` 本质上是
  [
  gcd(f(x),f(x+1))
  ]

这个关系有助于后续继续研究 `g(x)` 和 `q(x)` 的规律，但目前不能直接假设有简单闭式。

---

### 3. `b` 不是必须变量

`b` 的数学意义是按钮矩阵第一行，满足

[
b=Kp=Kc
]

其中 `K` 是 Krylov 矩阵。
但现在求解主线已经不再依赖：

[
b \to p \to q,g
]

而是直接走：

[
c \to gcd(f,c) \to q,g
]

因此：

* `b` 在当前求解流程中已经不是必要变量
* 如果只是为了最终求解，可以删掉
* 若以后真要恢复 `b`，也可以由 `c` 通过 `Kc` 算出

---

### 4. `k` 不是必须变量

当前 `k[j]` 的本质是：

[
k_j = H^j e_0
]

原代码里 `k` 的主要用途，是构造 `d[0]`：

[
d_0 = \sum_j g_j k_j = g(H)e_0
]

所以完全没必要先保存整张 `k` 表。
可以直接像计算 `z=q(H)y` 一样，从单位向量 `e0` 出发，做一次统一的“多项式作用在向量上”的运算：

[
ApplyPoly(a,v)=a(H)v
]

于是：

* `z = ApplyPoly(q, y)`
* `d0 = ApplyPoly(g, e0)`

这样 `k` 可以整张删掉。

---

## 三、建议先做的“第一阶段优化”（尽量少改结构，先简化代码）

---

### 1. `p` 省去（已确认可做）

#### 目标

删掉 `p` 的定义和计算，直接用 `c`。

#### 做法

把原来类似：
[
p=f\oplus c,\quad gcd(f,p)
]
改成：
[
gcd(f,c)
]

#### 收益

* 减少一个局部向量 / 中间量
* 数学上更直接
* 后续所有推导都统一围绕 `f,c,g,q`

---

### 2. `b` 省去

#### 目标

彻底删除 `b` 相关计算和存储。

#### 原因

`b` 不参与当前主求解链，只是旧版本遗留出来的“按钮矩阵第一行显式表达”。

#### 收益

* 删掉一整张 `TMat`
* 代码状态更少
* 不影响当前求解结果

---

### 3. `k` 省去

#### 目标

删除整张 `k` 表，改为直接计算 `g(H)e0`。

#### 做法

把
[
d_0=\sum_j g_j k_j
]
改成：
[
d_0 = ApplyPoly(g, e_0)
]

也就是：

* 初始化 `cur = e0`
* 若 `g[j]=1`，就把当前 `cur` 异或到 `d0`
* 再令 `cur <- H cur`
* 重复直到 `deg g`

#### 收益

* 删掉一整张 `k:TMat`
* 数学逻辑统一成一个原语：
  [
  ApplyPoly(a,v)=a(H)v
  ]
* 之后 `q(H)y` 和 `g(H)e0` 变成同一类操作

---

### 4. `z=q(H)y` 优化（如果当前向量全 0，则提前停止）

#### 目标

在计算
[
z=q(H)y
]
时，不要无脑跑满所有次数。

#### 原理

设当前迭代向量为 `cur`。若某一步出现：

[
cur = 0
]

则之后继续乘 `H` 仍然是 0：

[
H\cdot 0 = 0
]

因此后面所有项也都会是 0，可以立即停止。

#### 收益

* 在某些 `n` 上会明显减少无用迭代
* 这是严格正确的提前终止，不是近似剪枝

#### 同样适用于

[
d_0=g(H)e_0
]
的计算，因为也是同样的 `ApplyPoly` 模板。

---

### 5. 空间优化：把整张递推表改成滚动行

#### 目标

不要把 `f,c,l,l0,t` 全部保存成整张 `n×n` 矩阵。

#### 可滚动的对象

这些都只依赖前两行 / 当前行：

* `f`
* `c`
* `l`
* `l0`
* `t`

#### 做法

统一改成：

* `prev2`
* `prev1`
* `cur`

三行或两行缓冲即可。

#### 收益

如果最终结果 `t` 允许边生成边输出，那么求解器内部工作空间可从：

[
O(n^2)
]

降到接近：

[
O(n)
]

更严谨地说：

* 若完整 `t` 不常驻内存，只流式输出，则工作空间可做到 `O(n)`
* 若最终仍把整个 `t` 留在内存中，则总空间仍是 `O(n²)`

---

## 四、针对 `D=g(H)` 的优化（第二阶段，仍然保留现有奇异分支思路）

---

### 6. `d` 省区：只存带状非零区，不存整张矩阵

#### 关键性质

若
[
D=g(H),\quad r=\deg g
]
则 `D` 是带宽不超过 `r` 的带状矩阵。

因为：

* (H^j) 只会传播到距离不超过 `j` 的位置
* 所以 (g(H)) 的非零元只能落在
  [
  |u-v|\le r
  ]

#### 做法

不要再存完整 `d[n][n]`，只保存每行有效窗口。

#### 收益

* 空间从 `O(n²)` 降到 `O(nr)`，甚至更低
* 后续回代时也不用整行扫

---

### 7. `d` 改算法：先生成 `d0`，再由递推生成后续行

#### 目标

`D=g(H)` 不要通过大表 + 中间矩阵绕出来。

#### 做法

1. 先用
   [
   d_0 = g(H)e_0
   ]
   算出第一行
2. 再利用和 `b/d` 同类的十字递推，逐行生成 `D`

#### 收益

* 不需要 `k`
* 不需要整张 `d` 预先构造完
* 可以边生成边用于后续回代

---

### 8. 回代改算法：利用带宽 `r=deg g`，只扫有效区间

#### 原问题

当前回代时，本来一行很多地方都是 0，但代码可能还在做整行异或。

#### 目标

把“整行瞎扫”改成“只扫带宽”。

#### 做法

对于第 `i` 行，只处理其有效非零窗口：
[
[\max(0,i-r),\ \min(n-1,i+r)]
]

#### 收益

单次回代从：

[
O(n)
]

降成：

[
O(r)
]

因此 singular 分支整体复杂度可从：

[
O(n^2)
]

优化到：

[
O(nr)
]

最坏 `r=O(n)` 时仍是 `O(n²)`，但很多情形会更快。

---

### 9. 优化反向迭代

#### 含义

当前奇异分支通常是从右往左做“反向叠加 / 消元”。

#### 可优化点

1. 只遍历真正可能出现主元的位置区间
2. 结合 `D` 的带宽，动态缩小右端有效范围
3. 若 `z` 在当前有效区之外全 0，可提前停止
4. 若某一段根据带状结构必然不受后续影响，则不再重复检查

#### 收益

* 减少很多无意义循环
* 仍保持原有回代思路，不需要一下子换成全新算法

---

## 五、时间复杂度层面的认识（要分“输出整盘”和“只求第一行”）

---

### 10. 时间复杂度优化：如果要输出整盘 `t`，总复杂度下界就是 `O(n²)`

因为输出本身就有 `n²` 个格子。

所以：

* 如果目标是“求完整按钮矩阵并输出”，那最终不可能渐近优于：
  [
  O(n^2)
  ]
* 真正能进一步追求 `n log n` 的，是：

  1. 只求第一行解
  2. 只判定有无解 / 求解空间维数
  3. 或把“求解内部”做到准线性，但最终输出整盘时仍回到 `O(n²)`

---

### 11. 优化欧几里得

#### 当前可做

先直接对 `f,c` 做扩展欧几里得，而不是 `f,p`。

#### 更进一步

如果以后要继续追求更快复杂度，可以考虑：

1. **half-gcd / divide-and-conquer extended gcd**
2. 配合快速多项式乘法

理论上，求 `g,q` 这部分可从当前的 `O(n²)` 进一步压到：

[
O(M(n)\log n)
]

若多项式乘法 `M(n)=O(n log n)`，则可做到：

[
O(n\log^2 n)
]

但这已经属于“高级代数算法替换”，不是第一步就要改的内容。

---

## 六、第三阶段：更进一步的数学重构（可选，先不急着上代码）

这一阶段属于“把奇异 / 非奇异分支彻底统一”的方向。

---

### 12. 更统一的商环写法

因为：

* `c(H)x=y`
* `F=K^{-1}`
* 在 `K/F` 坐标下，`H` 对应“乘以 `x`”

所以原向量方程可以等价改写成：

[
c(x),u(x)\equiv b(x)\pmod{f(x)}
]

其中：

* (u = F x)
* (b = F y)

---

### 13. 这样可以省掉 `D=g(H)` 和反向叠加法

设
[
g = gcd(f,c),\quad f=gh,\quad c=gc_1
]

若
[
b = g b_1
]
则原方程可化成：

[
c_1 u \equiv b_1 \pmod h
]

而扩展欧几里得给出的 `q'` 实际上就是 `c_1` 在模 `h` 下的逆：

[
q' c_1 \equiv 1 \pmod h
]

于是直接解得：

[
u = q' b_1 \pmod h
]

最后再变回：

[
x = K u
]

#### 这意味着

* 当前奇异分支里的 `D=g(H)` 可以整体删除
* 反向叠加 / 回代可以整体删除
* singular / nonsingular 逻辑统一为同一个商环求解框架

#### 但

这一步改动会比较大，建议放到“前两阶段稳定之后”再做。

---

## 七、建议按这个顺序继续改

### 第一批（低风险、收益高）

1. `p` 省去，全面用 `c`
2. `b` 删除
3. `k` 删除，改成 `ApplyPoly(g,e0)`
4. `z=q(H)y` 和 `g(H)e0` 都加“当前向量全 0 则提前停止”
5. `f,c,l,l0,t` 改滚动行

---

### 第二批（保留原奇异分支逻辑，但让它更像样）

6. `d` 改成带状存储
7. `d` 先算第一行，再递推生成
8. 回代只扫带宽 `r`
9. 反向迭代加提前停止 / 缩区间

---

### 第三批（真正做数学重构）

10. 把 `D=g(H)` + 反向叠加法整体换成商环解法
11. 若还想继续提速，再考虑 half-gcd / 快速多项式乘法 / 更快的 `K/F` 变换

---

## 八、几个可以直接记住的简短口号（方便另一个对话继续）

1. **`p` 不要了，统一用 `c`**
2. **`b` 不在主求解链上，可以删**
3. **`k` 不要整张表，直接算 `g(H)e0`**
4. **所有 `a(H)v` 统一成一个 `ApplyPoly(a,v)` 模板**
5. **`z=q(H)y` / `g(H)e0` 若当前向量全 0，可提前停**
6. **`f,c,l,l0,t` 都尽量改成滚动行**
7. **`D=g(H)` 是带宽 `r=deg g` 的带状矩阵**
8. **奇异分支回代不要整行扫，只扫有效带宽**
9. **若最终要输出整盘，时间复杂度下界就是 `O(n²)`**
10. **若只求第一行 / 有无解，后续才值得追 `n log n`**
11. **更高级的终点是：把奇异 / 非奇异都统一成商环求解**
